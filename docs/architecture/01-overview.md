## 2. 分层架构说明

### 2.1 人类交互层

人类与系统交互的界面层，支持多种交互模式。

| 组件 | 功能 | 技术选型 |
|------|------|----------|
| 飞书界面 | 日常任务分配、进度查询、即时对话 | 飞书开放API |
| 审核控制台 | 关键决策审核、权限管理、冲突仲裁 | Web (React) |
| 文档编辑器 | 规范编辑、知识图谱管理 | Markdown编辑器 |

### 2.2 协调与协商层

系统的"大脑"，负责理解意图、协商协作、学习进化。

| 组件 | 功能 | 实现方式 |
|------|------|----------|
| 意图理解引擎 | 解析自然语言意图，提取实体和关系 | Claude Code |
| 动态协商引擎 | 能力匹配、可行性评估、响应生成 | 自建引擎 |
| 学习与进化引擎 | 模式提取、规范进化、跨系统学习 | 自建系统 |

### 2.3 文档与知识层

系统的"记忆"，存储规范、状态和知识。

| 层级 | 内容 | 格式 |
|------|------|------|
| 全局层 | 愿景、规则、全局图谱 | Markdown + YAML + JSON |
| 子系统层 | 规范、契约、状态、知识 | Markdown + YAML + JSON |
| 共享层 | 模式库、模板、Schema | Markdown + JSON |

### 2.4 执行层

系统的"手脚"，负责实际执行任务。

| 组件 | 功能 | 技术选型 |
|------|------|----------|
| Agent运行时 | AI Agent执行环境 | Claude Code + cmux |
| 代码执行器 | 执行内置代码和脚本 | Node.js / Python |
| 外部服务连接 | 连接外部API和服务 | HTTP / SDK |

   │                           │◄──────────────────────────│
   │                           │                           │

### 4.2 Agent 间通信（cmux send + inbox/outbox + CronCreate）

三层机制配合实现 Agent 间通信：

```
Orchestrator                                  财务子系统
───────────────────────                      ──────────────────────
1. 写 msg_YYYYMMDD_HHMMSS.json
   到 inbox-finance/
2. cmux send --surface <id>
   "收到新消息，请读取 inbox"
                                             3. 收到注入文本，立即响应
                                             4. 读取并处理 inbox-finance/ 消息
                                             5. 写 reply_*.json 到 outbox-finance/
                                             6. 原消息移到 processed/
                                             7. cmux send 通知发送方
8. CronCreate 轮询（兜底，10分钟间隔）
   检查 outbox-finance/
9. 读取回复内容
```

- **cmux send** — 即时通知，在 cmux 终端内通过 `cmux send --surface <id>` 唤醒目标 Agent
- **inbox/outbox** — 结构化消息，通过文件传递完整 JSON
- **CronCreate** — 兜底机制，定期轮询检查遗漏消息

目录结构：

```
shared/messages/
├── inbox-{agent}/      ← 各 Agent 的收件箱
├── outbox-{agent}/     ← 各 Agent 的发件箱（回复）
└── processed/          ← 已处理的消息归档
```

消息文件命名: `{type}_{YYYYMMDD_HHMMSS}.json`（如 `msg_20260409_103302.json`）

| | 注册表 | JSON | 文件存储 |

| | 代码执行 | Node.js / Python | 按需选择 |

## 6. 与现有架构的对比

| 方面 | 现有架构 (requirements.md) | 新架构 |
|------|---------------------------|--------|
| **系统边界** | 代码+文件夹定义 | 文档定义 |
| **Agent角色** | 任务执行器 | 自主决策体 |
| **交互方式** | 固定API/命令 | 动态协商 |
| **部门定义** | 固定层级结构 | 灵活子系统 |
| **知识管理** | 无 | 知识图谱 |
| **学习能力** | 无 | 持续学习 |
| **演进方式** | 人工更新 | 独立演进 |
