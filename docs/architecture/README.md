## 概述

本架构文档描述了一个**文档驱动的Agent协作生态系统**，核心理念是：

- **系统边界由文档定义**，而非代码
- **Agent具有自主决策能力**，在框架约束下独立运作
- **动态协商机制**，而非固定API
- **独立演进与自动适应**，形成有机生态系统

## 核心概念

### 范式转变

| 维度 | 传统范式 | 新范式 |
|------|----------|--------|
| **系统边界** | 代码定义 | 文档定义 |
| **Agent角色** | 任务执行器 | 自主决策体 |
| **交互方式** | 固定API | 动态协商 |
| **演进模式** | 中心化更新 | 独立演进+自动适应 |
| **知识管理** | 数据库存储 | 知识图谱分层 |

### 设计原则

1. **文档优先** - 先建立文档体系，再实现Agent
2. **自主决策** - Agent在约束边界内自主决策
3. **动态协商** - 通过意图理解进行交互协商
4. **持续进化** - 通过学习机制不断优化

## 快速开始

### 目录结构

```
company-ops/
├── CHARTER.md              # 愿景/使命/价值观
├── CONSTITUTION.yaml       # 核心规则
├── global-graph.json       # 全局知识图谱
├── subsystems/             # 子系统目录
│   ├── _registry.json      # 注册表
│   ├── task-system/        # 任务管理子系统 (Rust + SQLite)
│   ├── 财务/               # 财务管理子系统
│   ├── 法务/               # 法务合规子系统
│   └── ...
├── shared/
│   └── messages/           # Agent 间通信 (inbox/outbox)
│       ├── inbox-*/        # 各 Agent 收件箱
│       ├── outbox-*/       # 各 Agent 回复箱
│       └── processed/      # 已处理消息归档
└── orchestrator/
    ├── logs/               # 监控日志
    └── reports/            # 生成的报告
```

### 核心组件

- **文档层**: 规范、契约、状态文件
- **知识图谱**: 实体、关系、视图
- **协商引擎**: 意图理解、能力匹配、可行性评估
- **学习系统**: 模式提取、规范进化、跨系统学习
